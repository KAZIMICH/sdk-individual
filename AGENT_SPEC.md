# Agent Spec

## Текущий контекст

Реализованы создание КП и сценарии ассистента: чат, голосовой ввод, открытие почты, поиск и сайты белого списка. Активный следующий этап не выбран.

Перед изменениями источники истины: `PLAN.md`, `PROJECT_BRIEF.md`, затем `README.md` и существующий код. Работать только по подтверждённому пользователем пункту плана; отмечать `X` после его подтверждения.

## Рабочие правила

- Не добавлять лишний функционал, промежуточные файлы или абстракции без необходимости.
- Чётко разделять слои: backend — `app/` (кроме `app/web/`) и `scripts/`; frontend — `app/web/`.
- Frontend взаимодействует с backend исключительно по HTTP и не обращается к SQLite напрямую.
- Команды, их варианты, разрешённые действия и сайты хранятся в SQLite. URL не дублируются в JavaScript.
- Изменения схемы SQLite выполняются миграцией с сохранением существующих данных; повторный `python -m scripts.init_db` не создаёт дубликаты.
- После backend-изменений перезапускать сервер; после frontend-изменений обновлять страницу через `Ctrl+F5`.

## Проверки

```powershell
python -m unittest discover -s tests -v
node --check app\web\static\app.js
```

Ручная проверка обязательна для сценариев чата, голоса, открытия новой вкладки и fallback-ссылки.

## Границы

- Не читать и не отправлять почту.
- Не выполнять команды ОС и не применять browser automation.
- Не открывать произвольные URL: только адреса, разрешённые текущим сценарием и полученные через API.
- Не читать и не анализировать содержимое сайтов или поисковую выдачу.
- Не реализовывать напоминания: Flow исключён решением пользователя.
