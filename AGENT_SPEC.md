# Agent Spec

> Текущий scope агента: создание `.docx` по заранее сохраненным шаблонам. Остальные сценарии отложены.

## Роль агента
Приложение помогает пользователю создавать и сохранять типовые Word-документы через web-интерфейс.

## Цель агента
Ускорять создание типовых документов по единственному шаблону и сохранять результаты в БД и папке заказчика.

## Как реализован агент
Логика реализована в `FastAPI`: она проверяет данные формы, создаёт запись документа в SQLite, готовит папку и путь файла, формирует `.docx` и сохраняет его в папке заказчика. HTTP API отдаёт справочники, список КП с поиском и запускает генерацию; frontend обращается к нему по HTTP.

## Разрешенные tools
- Выбор заказчика и объекта проекта из независимых списков БД.
- Автоматическая подстановка выбранных данных в форму.
- Сохранение записи документа, связывающей выбранные заказчика и объект.
- Подстановка значений в `.docx`-шаблоны через `docxtpl`.
- Создание папки заказчика и сохранение документа в ней.

## Запрещенные действия
- Доступ к локальной файловой системе пользователя.
- Управление desktop-приложениями.
- Работа с содержимым почты.
- Отправка писем.
- Произвольные команды ОС.
- Browser automation.
- Самостоятельный запуск действий вне явно разрешенного сценария.
- Открытие ресурсов, web-поиск и голосовой ввод на текущем этапе.

## Разрешенные источники
- Список шаблонов документов, сохраненный в приложении.
- Данные заказчиков, объектов проекта и документов, сохраненные в БД.
- Строки состава работ, связанные с конкретным документом.
- Данные, введенные пользователем через web-интерфейс.

## Политика отказа
Если запрос выходит за пределы создания документов по шаблону, приложение должно сообщить, что в текущем этапе поддерживаются только работа с подготовленным шаблоном, SQLite и генерацией `.docx`.

## Поведение при недостатке данных
Если для выполнения команды не хватает данных, ассистент задает уточняющие вопросы и не выполняет действие до получения ответа.

## Проверка scope
Ассистент считается вышедшим за scope, если:
- пытается получить доступ к запрещенным источникам;
- инициирует неразрешенные действия;
- выполняет команды, не входящие в список поддерживаемых сценариев;
- пытается заменить ограниченный набор сценариев произвольной автоматизацией.

## Принципы реализации
- Добавлять функциональность только для текущего подтверждённого пункта `PLAN.md` или когда она необходима для его корректной работы и проверки.
- Не создавать новые файлы без практической необходимости; использовать существующую структуру проекта и общепринятые возможности выбранного стека.
- Не добавлять дублирующую документацию или промежуточные слои. Например, HTTP API FastAPI документируется и проверяется средствами самого приложения (`/docs` и автотесты), если отдельный документ не требуется пользователю или проекту.
