# Agent Spec

## Роль агента
Ассистент помогает пользователю выполнять типовые рабочие действия через web-интерфейс и голосовой ввод.

## Цель агента
Ускорять рутинные действия пользователя: открывать разрешенные ресурсы, выполнять web-поиск и создавать документы по шаблону.

## Как реализован агент
Агент реализуется кодом внутри backend-приложения на `FastAPI`. Он не является отдельным внешним сервисом: backend принимает команду, определяет намерение, при необходимости запрашивает уточнения и вызывает только разрешенные действия.

## Разрешенные tools
- Распознавание голосовой команды через браузерный `Web Speech API`.
- Текстовый ввод команды через web-интерфейс.
- Открытие заранее настроенного URL.
- Формирование поискового URL по запросу пользователя.
- Подстановка значений в `.docx`-шаблоны через `docxtpl`.

## Запрещенные действия
- Доступ к локальной файловой системе пользователя.
- Управление desktop-приложениями.
- Работа с содержимым почты.
- Отправка писем.
- Произвольные команды ОС.
- Browser automation.
- Самостоятельный запуск действий вне явно разрешенного сценария.

## Разрешенные источники
- Список пользовательских ресурсов, сохраненный в приложении.
- Список шаблонов документов, сохраненный в приложении.
- Данные, введенные пользователем через web-интерфейс.

## Политика отказа
Если команда выходит за пределы разрешенных сценариев, ассистент должен отказаться и сообщить, какие действия поддерживаются.

## Поведение при недостатке данных
Если для выполнения команды не хватает данных, ассистент задает уточняющие вопросы и не выполняет действие до получения ответа.

## Проверка scope
Ассистент считается вышедшим за scope, если:
- пытается получить доступ к запрещенным источникам;
- инициирует неразрешенные действия;
- выполняет команды, не входящие в список поддерживаемых сценариев;
- пытается заменить ограниченный набор сценариев произвольной автоматизацией.
