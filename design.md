# Design: форма создания коммерческого предложения

## Назначение

Внутренний web-интерфейс для создания одного коммерческого предложения по шаблону. Пользователь выбирает заказчика и объект проекта, заполняет параметры и состав работ, затем создаёт запись документа.

В MVP нет отправки email, копирования номера, скачивания или открытия `.docx`.

## Каркас страницы

На широком экране: тёмная верхняя панель и светлая рабочая область без боковой навигации.

Стартовый экран — «Ассистент»: короткое приветствие, карточка чата текущей сессии, текстовое поле команды и быстрые подсказки. История чата не сохраняется после обновления страницы.

Под приветствием размещается ненавязчивое пояснение, что голосовой ввод использует тестовое встроенное распознавание речи браузера и в дальнейшем будет заменён более точной технологией.

Круглая кнопка `?` в верхней панели открывает модальное окно справки. В нём кратко описаны текстовый и голосовой ввод, озвучка ответов и ограничение тестового распознавания. Список доступных команд не дублируется в разметке: он загружается из SQLite через backend API.

Кнопка микрофона запускает голосовой ввод одним нажатием и останавливает его вторым. При записи она меняет состояние на заметное оранжевое. Если браузер не поддерживает голосовой ввод или пользователь запретил микрофон, чат показывает текстовое объяснение и сохраняет возможность ввести команду текстом. Переключатель «Озвучка» включён по умолчанию и действует только в текущей вкладке.

Верхняя панель содержит название сервиса слева и навигацию справа: «Ассистент», «Список КП» и «Создать КП». Это не вкладки внутри одной карточки, а переходы между экранами.

Рабочая область содержит:

1. Заголовок текущего экрана и короткое пояснение.
2. Основной контент экрана.

Экран «Создать КП» содержит ссылку «К списку КП», одну белую карточку формы и сообщение об общем результате отправки под кнопкой формы.

Форма делится на три смысловые группы:

- «Заказчик»: выбор заказчика и автоматически заполненное поле «Руководитель».
- «Объект проекта»: независимый выбор объекта и автоматически заполненное поле «IMO».
- «Параметры предложения»: тема, сумма, срок разработки, срок действия, состав работ.

Блок «Состав работ» содержит строки с полями «Наименование» и «Комментарий», кнопку «Добавить строку» и удаление строки. Основное действие одно: «Создать документ».

## Экран списка КП

Над списком располагаются заголовок «Список КП», primary-кнопка «Создать КП» и одно поле поиска с подписью «Поиск по заказчику или объекту». Поиск не имеет отдельной кнопки: запрос отправляется после короткой паузы ввода или по клавише Enter.

На широком экране список — таблица со столбцами: «№», «Заказчик», «Объект», «Тема», «Создан», «Статус». Поиск возвращает строку, если текст найден в названии заказчика **или** объекта; регистр не учитывается.

На узком экране каждая строка таблицы превращается в карточку: заказчик и статус расположены сверху, объект, тема и дата — ниже. Если результатов нет, вместо пустой таблицы показывается нейтральное сообщение «Ничего не найдено».

## Корпоративные токены

```css
:root {
    --corp-blue: #11325B;
    --corp-orange: #FF4100;
    --bg-main: #eef2f6;
    --bg-card: #ffffff;
    --bg-muted: #f1f3f6;
    --border-light: #d8dee9;
    --border-input: #b0bec5;
    --bg-input: #f5f7fa;
    --text-main: #1b1f24;
    --text-muted: #6b7280;
    --color-error: #b42318;
    --color-error-bg: #fef3f2;
    --color-error-border: #fecdca;
    --color-success: #027a48;
    --color-success-bg: #ecfdf3;
    --color-success-border: #abefc6;
    --radius: 14px;
    --radius-card: 18px;
    --radius-input: 14px;
    --radius-btn: 8px;
    --shadow: 0 4px 14px rgba(0, 0, 0, 0.08);
    --layout-width: 1600px;
    --container-padding-x: 20px;
}
```

`--corp-blue` используется для шапки и основной кнопки. `--corp-orange` — только для небольших акцентов, не для конкурирующей основной кнопки. Фон рабочей области — `--bg-main`; карточка формы — `--bg-card`.

## Типографика и отступы

- Системный стек: `Inter, Arial, sans-serif`; если Inter недоступен — `Arial, sans-serif`.
- Заголовок страницы: 28 px, `font-weight: 700`, цвет `--text-main`.
- Заголовок группы: 18 px, `font-weight: 600`.
- Текст полей и кнопок: 14–16 px; подписи и подсказки: 14 px.
- Базовый шаг отступов: 8 px. Между группами формы — 32 px, между полями — 16 px.
- Карточка имеет `--radius-card`, границу `1px solid var(--border-light)` и `--shadow`; вложенные карточки не используются.

## Правила компонентов

### Поля

- У каждого поля есть постоянная текстовая подпись над ним; placeholder не заменяет подпись.
- Поля выбора заказчика и объекта доступны для ввода. Руководитель и IMO подставляются в поля `readonly` на фоне `--bg-muted`.
- Обычный фон поля — `--bg-input`, граница — `--border-input`, радиус — `--radius-input`.
- В фокусе поле получает чёткую синюю обводку; ошибка не должна исчезать при фокусе.
- Ошибка показывается под конкретным полем красным текстом; поле получает `--color-error-border`.

### Кнопки

- «Создать документ» — единственная primary-кнопка: `--corp-blue`, белый текст, радиус `--radius-btn`.
- «Добавить строку» — secondary-кнопка: белый фон, синяя граница и текст.
- Удаление строки — компактная текстовая кнопка красного цвета; не является primary-действием.
- При отправке primary-кнопка отключена и содержит текст «Создание…». Другие элементы формы также не должны отправлять повторный запрос.

### Навигация, поиск и таблица

- Активный пункт «Создать КП» или «Список КП» отмечается оранжевой нижней границей; неактивный остаётся текстовой ссылкой без конкурирующего акцента.
- Поле поиска использует те же правила, что и поля формы.
- Заголовки таблицы используют `--bg-muted`; строки разделяются `--border-light`. Сортировка, фильтры и постраничная навигация не добавляются до появления необходимости.
- Статус показывается компактным бейджем: `pending` — нейтральный, `generated` — зелёный, `error` — красный.

### Сообщения

- Ошибки API, не привязанные к конкретному полю, показываются отдельным блоком `--color-error-bg` / `--color-error-border`.
- Успешная генерация показывается отдельным блоком `--color-success-bg` / `--color-success-border` с понятным текстом о создании документа и Word-файла.
- Не показывать технические детали SQLite, Python или HTTP-статусы пользователю.

## Состояния

- До загрузки справочников форма не отправляется; при ошибке загрузки показывается сообщение без технических деталей.
- Если справочник пуст, выбор соответствующего значения невозможен и серверная валидация не позволит создать документ.
- При ошибке отправки: отобразить ошибки полей из API; введённые пользователем значения сохранить в форме.
- После успешной генерации: показать success-блок, очистить форму и оставить одну пустую строку состава работ для следующего документа.
- При загрузке списка: показывать текст «Загрузка списка…» в области таблицы.
- При ошибке списка: показать сообщение и кнопку повторной попытки; не выдавать техническое описание ошибки.

## Адаптивность

- От 1024 px: форма использует две колонки там, где поля логично расположить парами; состав работ остаётся одной колонкой.
- Меньше 1024 px: поля формы переходят в одну колонку.
- Меньше 768 px: основной контейнер использует `--container-padding-x`, primary-кнопка растягивается на всю доступную ширину.
- Интерактивные элементы имеют высоту не менее 40 px.

## Ограничения

- Не использовать градиенты, стеклянные эффекты, лишние вложенные карточки и несколько конкурирующих primary-кнопок.
- Не добавлять UI для функций вне MVP.
- Frontend получает данные и создаёт документ только через HTTP API backend.
