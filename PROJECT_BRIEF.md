# Project Brief

> Текущий этап завершён: реализованы создание КП и пять сценариев рабочего ассистента. Новый сценарий пока не выбран.

## Назначение

«Рабочий ассистент» — локальное web-приложение для типовых рабочих действий. Базовый сценарий — создание коммерческого предложения в Word по шаблону. Ассистент дополняет его ограниченными командами, доступными текстом и голосом.

## Реализованные пользовательские сценарии

### Flow 1: создание КП

Пользователь выбирает заказчика и объект, заполняет форму и состав работ. Приложение проверяет данные, сохраняет запись в SQLite и создаёт `.docx` по шаблону. Список КП поддерживает поиск по заказчику или объекту.

### Flow 2: чат и голосовой ввод

Пользователь вводит команду в чат или произносит её после нажатия кнопки микрофона. Ассистент отвечает текстом и голосом. История существует только в текущей странице; распознавание речи использует тестовую технологию браузера.

### Flow 3: открытие почты

По настроенной команде открывается `https://e.mail.ru/`. Приложение не получает доступ к содержимому почты и не отправляет письма.

### Flow 4: поиск в интернете

Команда вида «Найди <запрос>» открывает Яндекс Поиск с URL-кодированным запросом. Приложение не запрашивает, не читает и не анализирует поисковую выдачу.

### Flow 5: открытие сайтов

По настроенной команде открывается сайт из белого списка: Российский морской регистр судоходства, Adomat Marine, Российское классификационное общество (РКО) или Fleetphoto. Понятная команда выполняется сразу; произвольные URL не поддерживаются.

## Данные и архитектура

- SQLite хранит справочники, КП, строки работ, команды ассистента, разрешённые действия и сайты белого списка.
- Backend — FastAPI, SQLite, генерация Word и HTTP API.
- Frontend — HTML, CSS и JavaScript в `app/web/`; доступ к данным только через HTTP API.
- Сумма округляется до целого рубля по правилу `0,5` вверх. Суммы и сроки выводятся цифрами и прописью с русскими склонениями.

## Ограничения

- Нет доступа к содержимому почты и отправки писем.
- Нет команд ОС, browser automation и открытия произвольных адресов.
- Нет чтения или анализа внешних сайтов и поисковой выдачи.
- Нет открытия сгенерированных документов для редактирования в приложении.
- Flow с напоминаниями исключён из развития проекта решением пользователя.

## Запуск и проверка

Реальные команды запуска и проверки приведены в `README.md`. Перед запуском необходимо выполнить `python -m scripts.init_db`, чтобы применить миграции SQLite и добавить начальные настройки команд.
