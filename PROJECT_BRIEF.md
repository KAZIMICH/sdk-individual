# Project Brief

## Business Logic
Пользователь обращается к web-ассистенту голосом или текстом. Ассистент распознает команду, определяет тип сценария и выполняет разрешенное действие: открывает ресурс, выполняет поиск или создает документ по шаблону.

Для документных сценариев ассистент собирает недостающие данные через уточняющие вопросы, подставляет значения в шаблон и отдает готовый `.docx` для скачивания.

## Пользовательские flow
### Flow 1: создание документа
1. Пользователь вводит команду `Создай КП`.
2. Ассистент определяет шаблон документа.
3. Ассистент задает недостающие вопросы.
4. Пользователь отвечает.
5. Ассистент генерирует готовый `.docx`.

### Flow 2: открытие ресурса
1. Пользователь вводит `Открой почту`.
2. Ассистент находит заранее настроенный ресурс.
3. Ассистент открывает URL в браузере.

### Flow 3: поиск
1. Пользователь вводит `Найди <запрос>`.
2. Ассистент формирует поисковый URL.
3. Ассистент открывает страницу поиска.

## Входные данные
- голосовая команда пользователя;
- текстовая команда пользователя;
- тип документа;
- значения для плейсхолдеров шаблона;
- список настроенных ресурсов;
- список шаблонов документов.

## Выходные данные
- открытый web-ресурс;
- открытая поисковая страница;
- готовый `.docx` по шаблону;
- текстовый ответ ассистента;
- история команд.

## Ограничения MVP
- только ограниченный набор разрешенных команд;
- только заранее настроенные ресурсы и шаблоны;
- только web-формат взаимодействия;
- без доступа к локальной файловой системе;
- без desktop automation;
- без работы с содержимым почты;
- без отправки писем;
- без browser automation.

## Примеры пользовательских запросов
- `Открой почту`
- `Найди Python`
- `Создай отчет`
- `Создай КП`
- `Создай протокол встречи`
