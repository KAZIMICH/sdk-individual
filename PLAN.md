## 1.1 Целевой сценарий

1. Пользователь вводит или произносит команду открытия почты.
2. Frontend сопоставляет её с данными, полученными из backend API.
3. Только команда с разрешённым действием `open_mail` получает URL из SQLite.
4. Ассистент сообщает об открытии почты и открывает `https://e.mail.ru/` в новой вкладке; при блокировке всплывающего окна показывает безопасную ссылку для ручного перехода.
5. Приложение не получает доступ к почтовому ящику и не выполняет иных действий.

## 2. Требования к результату

- Новое действие и его синонимы хранятся в SQLite, без списка команд и адресов в JavaScript.
- Изменение текущего ограничения `assistant_command_aliases` выполняется миграцией с сохранением команд создания КП.
- URL почты хранится в таблице разрешённых действий и связан с кодом `open_mail`.
- API возвращает frontend только активные команды и их разрешённые данные.
- Frontend поддерживает только известные коды действий: `create_document` и `open_mail`; значение из БД не может запустить команду ОС или произвольный сценарий.
- Автотесты покрывают миграцию, выдачу API, существующие команды КП и открытие почты на уровне frontend-логики. Перед завершением обязательна ручная проверка текста, голоса и блокировки popup.

## 6. Этап 11 — поиск в интернете

### 6.1 Целевой сценарий

1. Пользователь вводит или произносит команду, например «Найди правила классификации судов».
2. Frontend выделяет команду поиска и текст запроса по данным из SQLite.
3. При отсутствии запроса ассистент просит уточнить, что искать.
4. При наличии запроса frontend открывает страницу Яндекса с URL-кодированным запросом в новой вкладке и оставляет fallback-ссылку в чате.
5. Пользователь сам выбирает результаты и работает с открытой поисковой страницей.

Допущение: на первом шаге используется Яндекс с базовым URL `https://yandex.ru/search/?text=`. URL хранится в SQLite и может быть изменён без правки JavaScript.

### 6.2 Требования и границы

- В SQLite добавляется разрешённое действие `web_search` с признаком, что ему нужен текст запроса.
- Команды поиска и их синонимы хранятся в SQLite; запрос пользователя не сохраняется.
- Frontend обрабатывает только известные действия `create_document`, `open_mail` и `web_search`.
- Поиск открывает только сформированный URL настроенного поисковика; нет HTTP-запросов к поисковику из backend, парсинга выдачи, скрейпинга или ответов ассистента по найденным данным.
- Существующие команды КП и почты не изменяются.
- Автотесты покрывают миграцию, seed, API и обработку отсутствующего/непустого поискового запроса; перед завершением обязательна ручная проверка текста, голоса и fallback-ссылки.

## 7. Этап 12 — открытие сайтов из белого списка

### 7.1 Целевой сценарий

1. Пользователь вводит или произносит понятную команду открытия сайта, например «Открой Российский морской регистр судоходства».
2. Frontend сопоставляет фразу с активным псевдонимом из данных backend API.
3. Для распознанной команды ассистент сразу сообщает об открытии и открывает только связанный с ней сайт в новой вкладке.
4. Если браузер заблокировал всплывающее окно, в чате остаётся безопасная ссылка для ручного перехода.
5. Неизвестная фраза не открывает произвольный URL и получает обычный текстовый ответ ассистента.

Первичный белый список: Российский морской регистр судоходства (`https://rs-class.org/`), Adomat Marine (`https://adomatmarine.ru/`), Российское классификационное общество (РКО) (`https://rfclass.ru/`), Fleetphoto (`https://fleetphoto.ru/`).

### 7.2 Требования и границы

- Сайты, их URL, активность и командные варианты хранятся в SQLite; в JavaScript нет жёстко заданного списка или URL.
- Для сайта используется отдельная сущность в БД, а командные псевдонимы связываются с ней. Это позволяет добавлять сайт, отключать его и расширять набор фраз без изменения frontend-кода.
- Backend отдаёт только активные команды и активные сайты из белого списка.
- Frontend поддерживает только известный код действия `open_site`; URL валидируется как `https` перед открытием.
- Понятная команда выполняется сразу, без подтверждения. Уточнение синонимов и нераспознанных фраз — технический долг, не входит в минимальную версию.
- Не входят в этап: открытие произвольных адресов, переход по страницам сайта от имени пользователя, чтение или анализ содержимого сайтов, экран администрирования настроек.
