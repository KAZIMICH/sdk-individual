# Plan

## Этап 1. Согласование идеи
Результат: зафиксированы идея проекта, пользователь, бизнес-логика, стек, граница MVP, риски и вопросы к наставнику.

## Этап 2. Проектирование структуры
Результат: определены сценарии команд, структура шаблонов документов, хранение настроек и пользовательских ресурсов.

## Этап 3. Реализация backend
Результат: поднят `FastAPI`-проект, реализованы API для команд, шаблонов и генерации документов.

## Этап 4. Реализация web-интерфейса
Результат: добавлен интерфейс для голосового и текстового ввода, настройки ресурсов и просмотра истории команд.

## Этап 5. Генерация документов
Результат: подключен `docxtpl`, реализована подстановка данных в `.docx`-шаблоны и выдача готового файла.

## Этап 6. Проверка MVP
Результат: проверены ключевые сценарии `Открой почту`, `Найди <запрос>`, `Создай <документ>`.
