# Рабочий ассистент

Локальное web-приложение для повседневных рабочих задач. Оно создаёт коммерческие предложения в Word и выполняет ограниченный набор текстовых и голосовых команд.

## Возможности

- Создание КП по форме, сохранение в SQLite и генерация `.docx` по шаблону.
- Список сохранённых КП с поиском по заказчику или объекту.
- Чат с текстовым и голосовым вводом, а также озвучкой ответов.
- Открытие настроенного почтового ящика.
- Открытие Яндекс Поиска с переданным запросом.
- Открытие сайтов из настроенного белого списка.
- Справка по кнопке `?` с актуальными командами из SQLite.

История чата существует только до обновления страницы.

## Команды ассистента

Команды и их варианты хранятся в SQLite. Полный актуальный список доступен в справке по кнопке `?`.

Примеры:

- `Создай КП` — открыть форму создания КП.
- `Открой почту` — открыть настроенный почтовый ящик.
- `Найди правила классификации судов` — открыть страницу поиска с запросом.
- `Открой РС`, `Открой РМРС` — открыть сайт Российского морского регистра судоходства.
- `Открой РКО` — открыть сайт Российского классификационного общества.
- `Открой Adomat Marine`, `Открой Fleetphoto` — открыть соответствующий сайт из белого списка.

Команду можно ввести в чат или произнести после нажатия кнопки микрофона.

> Голосовой ввод использует тестовое встроенное распознавание речи браузера, поэтому его качество может быть нестабильным. Если браузер заблокирует открытие новой вкладки, ассистент оставит ссылку для ручного перехода.

## Запуск

Нужен Python 3.13 или совместимая версия Python 3.

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

Создайте `.env` по примеру `.env.example`. Если настройки по умолчанию подходят, значения можно не заполнять.

Инициализируйте или обновите SQLite-базу:

```powershell
python -m scripts.init_db
```

Команда безопасна для повторного запуска: она применяет миграции и добавляет только отсутствующие начальные записи.

Запустите приложение:

```powershell
python app/main.py
```

Откройте [http://127.0.0.1:8000/](http://127.0.0.1:8000/).

После изменения backend перезапустите сервер. После изменения frontend обновите страницу через `Ctrl+F5`.

## Проверки

```powershell
python -m unittest discover -s tests -v
node --check app\web\static\app.js
```

## Границы приложения

- Backend: `app/` (кроме `app/web/`) и `scripts/`; frontend: `app/web/`.
- Frontend взаимодействует с backend только по HTTP.
- Приложение не читает и не отправляет письма, не выполняет команды ОС и не открывает произвольные адреса.
- Поиск открывает поисковую страницу, но не читает и не анализирует выдачу.
- Сайты открываются только из белого списка, хранящегося в SQLite.
