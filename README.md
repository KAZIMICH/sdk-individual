# Voice Work Assistant

Web-ассистент с поддержкой голосового ввода для выполнения типовых рабочих действий в одном интерфейсе.

## Пользователь и задача
Пользователь - сотрудник, который регулярно выполняет повторяющиеся действия в браузере и создает типовые документы.

Задача - ускорить рутинные действия: открыть рабочий ресурс, найти информацию или создать документ по шаблону.

## Основной сценарий
Пользователь открывает web-интерфейс, вводит команду голосом или текстом, ассистент распознает намерение, задает уточняющие вопросы при необходимости и выполняет разрешенное действие.

Основной сценарий MVP - создание документа по шаблону, например `Создай КП`, с последующей генерацией готового `.docx` для скачивания.

## Выбранный стек
- `Python 3.13`
- `venv`
- `FastAPI`
- `SQLite`
- `JSON`
- `Markdown`
- `Web Speech API`
- `docxtpl`

## Почему выбран именно этот стек
Стек простой, локальный и проверяемый в рамках курса. `FastAPI` подходит для web-интерфейса и API, `SQLite` и `JSON` закрывают хранение настроек и шаблонов без лишней инфраструктуры, `docxtpl` позволяет подставлять данные в шаблоны `.docx`, а `Web Speech API` дает голосовой ввод прямо в браузере.

## Какие технологии из курса используются
- `Python`
- `venv`
- `SQLite`
- `JSON`
- `Markdown`

## Какие технологии вне курса нужны и зачем
- `FastAPI` - для реализации web-интерфейса и backend-логики.
- `Web Speech API` - для голосового ввода в браузере.
- `docxtpl` - для генерации документов из `.docx`-шаблонов с плейсхолдерами `{{...}}`.

## Технические риски
- Поддержка `Web Speech API` зависит от браузера и его настроек.
- Голосовой ввод может быть нестабилен, поэтому нужен текстовый fallback.
- `docxtpl` требует аккуратной структуры шаблонов.
- Проект не должен уходить в доступ к локальной файловой системе, desktop automation или browser automation.
- Внешние интеграции кроме открытия URL в браузере в MVP не используются.

## Как проект будет запускаться
Локально, через Python-окружение и запуск `FastAPI`-приложения.

## Артефакты на выходе
- web-интерфейс ассистента;
- готовый `.docx` по шаблону;
- история выполненных команд.

## Что точно не входит в MVP
- доступ к локальной файловой системе пользователя;
- управление desktop-приложениями;
- работа с содержимым почты;
- отправка писем;
- произвольные команды ОС;
- browser automation;
- регистрация, авторизация, роли;
- редактирование документов после генерации внутри приложения.
