# Word Document Generator

Веб-приложение для создания и сохранения типовых Word-документов по шаблону.

> Текущий фокус разработки: генерация Word-документов (`.docx`) по шаблонам. Сценарии открытия ресурсов, web-поиска и голосового ввода отложены.

## Пользователь и задача
Пользователь - сотрудник, который регулярно выполняет повторяющиеся действия в браузере и создает типовые документы.

Задача текущего этапа — создавать типовые документы по шаблону и сохранять готовый `.docx` внутри приложения.

## Основной сценарий
Пользователь выбирает заказчика и объект проекта из независимых списков БД, заполняет форму, а приложение сохраняет запись документа, формирует и сохраняет готовый файл в папке заказчика.

Основной сценарий MVP — создание одного типа документа по шаблону с последующим сохранением `.docx` в папке выбранного заказчика.

## Выбранный стек
- `Python 3.13`
- `venv`
- `FastAPI`
- `SQLite` и встроенный модуль `sqlite3`
- `Markdown`
- `docxtpl`

## Почему выбран именно этот стек
Стек простой, локальный и проверяемый в рамках курса. `FastAPI` подходит для web-интерфейса и API, SQLite хранит заказчиков, объекты, документы и состав работ, а `docxtpl` подставит данные в `.docx`-шаблон.

## Какие технологии из курса используются
- `Python`
- `venv`
- `SQLite`
- `Markdown`

## Какие технологии вне курса нужны и зачем
- `FastAPI` - для реализации web-интерфейса и backend-логики.
- `docxtpl` - для генерации документов из `.docx`-шаблонов с плейсхолдерами `{{...}}`.

## Технические риски
- `docxtpl` требует аккуратной структуры шаблонов.
- Данные заказчиков и объектов являются независимыми справочниками; их связь создается только в записи документа.
- Имена папок и файлов должны быть безопасны для Windows и Linux и не должны перезаписывать существующие документы.
- Пути к папке документов задаются в конфигурации приложения.
- SQLite-файл и сгенерированные документы являются локальными данными приложения и не должны попадать в Git.

## Развертывание и запуск

Проект рассчитан на локальный запуск на Windows и Linux с Python 3.13.

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте виртуальное окружение: `python -m venv .venv`.
3. Активируйте его:
   - Windows PowerShell: `.\.venv\Scripts\Activate.ps1`;
   - Linux: `source .venv/bin/activate`.
4. Установите зависимости: `python -m pip install -r requirements.txt`.
5. Создайте файл `.env` по примеру `.env.example` и при необходимости настройте пути к БД, Word-шаблонам и папке результатов.
6. Инициализируйте БД и вымышленные тестовые справочники: `python -m scripts.init_db`.
7. Запустите приложение: `uvicorn app.main:app --host 127.0.0.1 --port 8000`.

Пункт 6 уже доступен. Пункт 7 станет доступен после подключения и запуска FastAPI на текущем этапе 3. После сквозной проверки версии зависимостей будут зафиксированы в `requirements.txt`.

## Артефакты на выходе

- запись документа в БД;

- управление desktop-приложениями;
- работа с содержимым почты;
- отправка писем;
- произвольные команды ОС;
- browser automation;
- регистрация, авторизация, роли;
- редактирование документов после генерации внутри приложения.
- скачивание и открытие документов для редактирования;
- открытие ресурсов и web-поиск на текущем этапе;
- голосовой ввод на текущем этапе.
